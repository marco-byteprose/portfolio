Cocktail Connoisseur - The Bar
Cocktail recipes website

MongoDB Express React NodeJS Docker

Summary

Project Goal

I am an amateur home cook and have learned that a base recipe makes experimentation in the kitchen easier and much more fun. However, when I'm hosting dinner parties, I find I spend more time searching and skippping through food blogs to find the cocktail recipe a guest requested than spending time with my guests. Even more so when there are different cocktail requests.

This site gives me standard recipes that I can adjust when needed. And I don't have to filter through modified recipes or thousands of popup ads, saving me a lot of time, an essential factor when hosting.

Technologies Used

The dataset is stored on a MongoDB database where each document holds the details for each cocktail: recipe, ingredients, garnish, glass, and more. The backend is an API that communicates with the database and is built with Node.js and Express. For the client, the React library is used with custom hooks. And finally, for deployment, the application is containerized with Docker and deployed on a secure virtual private server.

UX Case Study

User Research Prototype Usability Testing

Summary

Problem

There are many variables when it comes to choosing plants for a specific garden space, and personally it can be overwhelming at times. I aim to develop a comprehensive solution for garden enthusiasts and amateurs seeking tailored native plant recommendations optimized for their garden spaces.

This solution will account for factors such as flowering patterns, soil type, sunlight requirements, companion planting, culinary & medicinal usages, and pollinators. By offering personalized guidance, users will be empowered to create thriving gardens that align with their goals and local ecosystem.

Research and Discovery

A standard script was developed to interview target users. The gathered data was analyzed and used to perform a task analysis and to determine design requirements. After which, personas were created to guide UI decisions based on these findings.

Design Process

I initiated the design process by conducting a hierarchical task analysis to dissect user workflows, followed by the creation of user stories to capture essential user needs and goals. Utilizing Figma, I developed a low-fidelity interactive prototype to visualize design concepts and functionality. Then, I prepared a usability testing script with a brief scenario and crafted specific questions and metrics for evaluation. Usability testing sessions were conducted with three target users, yielding valuable feedback.

Based on the evaluation findings, I iteratively revised the design to enhance utility, usability, and overall user experience. These revisions were informed by user insights and aimed at addressing identified pain points and usability issues effectively. The design evolution culminated in the development of a final hi-fidelity prototype using Figma, reflecting optimized design elements and incorporating user-tested improvements.

DonationPal
A GoFundMe clone

MongoDB Express React NodeJS Google Cloud Platform

Summary

Project Goal

This project focuses on authentication, authorization, and an integrated payment system with Stripe.

Technologies Used

DonationPal employs the MERN stack. The MongoDB database holds several collections: users, campaigns, and donations. JSON Web Token (JWT) is used for user authentication and authorization. For donations, the Stripe API is used to securely transfer funds.

The application is deployed on Google App Engine from the Google Cloud Platform as two services: the client as the default, and the backend as the api.
